# Gemini Styleguide for Totem Playground

This document defines the architectural patterns, styling, and coding conventions for the `totem-playground` repository. The Gemini Code Assist bot will use this to enforce rules during PR reviews.

## 1. Architectural Boundaries

- **Strict Typing:** Avoid `any`. Use generics and strict interfaces.
- **Modularity:** Ensure separation of concerns between components, API routes, and database abstraction layers.

## 2. Naming Conventions

- **Clear and Descriptive:** Use descriptive variable names rather than metaphors.
