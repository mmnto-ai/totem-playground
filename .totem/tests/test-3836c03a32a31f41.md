<!-- Never build SQL queries with string concatenation -->

## Should fail

```ts
// String concatenation with `+` — the classic SQL injection vector.
// User input flows directly into the query string, letting an attacker
// terminate the literal and append arbitrary statements.
const query = "SELECT * FROM users WHERE id = " + userId;
const user = await db.raw(query);
```

```ts
// Template literal with `${}` interpolation — the modern-JS flavor of
// the same bug. The backtick syntax looks safer than manual concatenation
// but the runtime result is identical: untrusted data is spliced into
// the query string at the moment of execution.
await db.raw(`UPDATE users SET email = '${newEmail}' WHERE id = ${userId}`);
```

## Should pass

```ts
// Parameterized query (pg / node-postgres style).
// The query literal is static; user input flows through the parameter
// array and the driver handles escaping + type coercion.
const result = await db.query(
  "SELECT * FROM users WHERE id = $1",
  [userId],
);
```

```ts
// Prepared statement (better-sqlite3 / mysql2 style).
// The statement is compiled once with a `?` placeholder, then runtime
// values are bound at execution time. The query string and user input
// never touch each other at the string level.
const stmt = db.prepare("SELECT * FROM users WHERE email = ?");
const user = stmt.get(email);
```

```ts
// ORM query (Prisma style).
// The `findMany` API builds the query through typed method calls; no
// raw SQL string exists on the call site at all, so there is nothing
// for user input to be concatenated into.
const users = await prisma.user.findMany({
  where: { id: userId },
});
```
